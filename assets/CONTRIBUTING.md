# Contributing

Thank you for your interest in Sovereign System Architecture.

This repository is maintained as a public showcase and governance layer. Contributions must preserve the public/private boundary and keep the project suitable for enterprise, procurement, and security review.

## Contribution Scope

Appropriate public contributions include:

- Documentation clarity improvements.
- Accessibility improvements for public static pages.
- Typo fixes and broken public link corrections.
- Non-sensitive governance improvements.
- Minor presentation improvements that do not change private product behavior.

Do not submit:

- Secrets, API keys, tokens, credentials, or private configuration.
- Customer data, client references, or environment-specific details.
- Operational deception logic, detection rules, telemetry schemas, or internal playbooks.
- Proprietary prompts, commercial strategy, pricing terms, or acquisition materials.
- Offensive security workflows, exploit instructions, or intrusive testing methods.

## Documentation Standards

Public documentation should be:

- Clear, factual, and procurement-safe.
- Defensive-security focused.
- Free from exaggerated security claims.
- Careful to distinguish public showcase material from private enterprise deliverables.
- Written in a premium, professional tone.

Avoid militarized, aggressive, or legally risky language. Use precise language that a CISO, legal reviewer, or enterprise buyer can evaluate safely.

## Pull Request Checklist

Before submitting a pull request, confirm that:

- The change is limited to public-safe content.
- No private implementation details are disclosed.
- No secrets or credentials are included.
- The public edition and private enterprise edition remain clearly separated.
- The change does not alter product logic or behavior unless explicitly requested by the maintainer.

## Maintainer Review

All contributions may be edited, rejected, or deferred at maintainer discretion. Commercial, private, or enterprise-specific requests should be handled through a separate written agreement.
