# Security Policy

## Supported Scope

This public repository contains static showcase material and governance documentation for Sovereign System Architecture. Security review for the public edition is limited to files that are visible in this repository.

The following areas are in scope for responsible disclosure:

- Public repository files that expose secrets or sensitive configuration.
- Public documentation that accidentally discloses private implementation details.
- Static page behavior that could create browser security concerns.
- Broken public links that could mislead users into unsafe or unofficial channels.

## Out of Scope

The following items are outside the scope of this public repository:

- Private enterprise transfer packages.
- Client deployments, customer environments, or internal infrastructure.
- Proprietary telemetry, detection, deception, or monitoring logic.
- Private prompts, playbooks, strategy documents, or commercial terms.
- Social engineering, denial-of-service testing, phishing, or intrusive scanning.

Do not attempt to access systems, accounts, data, or infrastructure that you do not own or have explicit written permission to test.

## Reporting a Vulnerability

Please report security concerns privately by email:

**contact@localpulse.pro**

Include:

- A concise description of the issue.
- The affected file, URL, or public artifact.
- Steps to reproduce when safe and non-intrusive.
- Potential impact.
- Suggested remediation, if known.

Please do not include secrets, customer data, private keys, tokens, exploit payloads, or sensitive third-party information in the report.

## Response Expectations

Security reports are reviewed on a best-effort basis. Confirmed public repository issues will be triaged according to severity, reproducibility, and exposure risk.

If an issue concerns a private enterprise package, a client environment, or non-public commercial material, it must be handled under the applicable private agreement and disclosure process.

## Safe Harbor

Good-faith, non-intrusive research focused on this public repository is welcome when it follows this policy. Activity that accesses private systems, disrupts services, extracts data, or targets third parties is not authorized.
